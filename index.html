Skill de Claude — Los 7 componentes

La diferencia
está en una
sola línea.

La mayoría construye skills con 3 componentes y se pregunta por qué Claude no les hace caso. Aquí están los 7 — y por qué el más ignorado es el que lo decide todo.

Los 7 componentes

01
Identidad

YAML Frontmatter

El carnet de identidad del skill. Le dice a Claude cómo se llama y cuándo debe cargarlo. Sin frontmatter claro, Claude ignora el skill por completo — aunque el resto esté perfecto.

02
El cerebro

Instructions Block

Le dices exactamente qué hacer y qué nunca hacer dentro de este skill. Cuanto más específico, menos margen para la improvisación.

03
El atajo

Examples Section

Muestra a Claude un input de ejemplo y el output exacto que esperas. Dos ejemplos bien elegidos reducen los errores a la mitad.

04
Contexto

Supporting Files

Plantillas, scripts, documentos de referencia. En lugar de pegar todo en un bloque, enlaza archivos externos. El skill se mantiene limpio y el contexto siempre actualizado.

★ El más ignorado
05
Trigger Rule

El componente que
decide si todo funciona.

Claude lee esta descripción para decidir si debe activar el skill o no. Si es vaga, lo ignora. Si es genérica, lo carga cuando no toca. Si es precisa, actúa exactamente cuando debe.

trigger_rule.md 
# ✗  Mal — demasiado vago
trigger: "usar cuando el usuario pida ayuda con marketing"

# ✓  Bien — preciso y accionable
trigger: |
  Activar cuando el usuario pida un post para LinkedIn.
  NO activar para emails, newsletters o Instagram.
  Señales clave: "LinkedIn", "post", "publicación".
06
Mata-errores

Negative Instructions

Dile a Claude qué no hacer. Esta sección previene el 80% de los outputs incorrectos. Si hay algo que Claude hace por defecto y tú no quieres, aquí es donde se corta.

07
El multiplicador

Composability

Los skills pueden trabajar juntos. Claude puede cargar varios a la vez. Un skill de voz + formato + canal = un sistema que trabaja como tú, no como Claude por defecto.
