Los 7 componentes de un Skill de Claude que realmente funciona

La mayoría usa solo 3. Aquí están los 7 — y por qué el que más se ignora es el que lo decide todo.

Scroll para ver los componentes

Los 7 componentes

01
Identidad

YAML Frontmatter

El carnet de identidad del skill. Le dice a Claude cómo se llama y cuándo debe cargarlo. Sin frontmatter claro, Claude ignora el skill por completo — aunque el resto esté perfecto.

02
El cerebro

Instructions Block

Le dices exactamente qué hacer y qué nunca hacer dentro de este skill. Cuanto más específico, menos margen para la improvisación.

03
El atajo

Examples Section

Muestra a Claude un input de ejemplo y el output exacto que esperas. Dos ejemplos bien elegidos reducen los errores a la mitad.

04
Contexto

Supporting Files

Plantillas, scripts, documentos de referencia. En lugar de pegar todo en un bloque, enlaza archivos externos. El skill se mantiene limpio y el contexto siempre actualizado.

05
El más ignorado

Trigger Rule

Claude lee esta descripción para decidir si debe activar el skill o no. Es el componente que más se ignora — y el que decide si todo lo demás funciona.

Si es vaga, Claude la ignora. Si es genérica, carga el skill cuando no toca. Si es precisa, actúa exactamente cuando debe.

trigger_rule · ejemplo 
# Mal — demasiado vago
trigger: "usar cuando el usuario pida ayuda con marketing"

# Bien — preciso y accionable
trigger: |
  Activar cuando el usuario pida un post para LinkedIn.
  NO activar para emails, newsletters o contenido para Instagram.
  Señales: menciona "LinkedIn", "post", "publicación profesional".
06
El mata-errores

Negative Instructions

Dile a Claude qué no hacer. Esta sección previene el 80% de los outputs incorrectos. Si hay algo que Claude hace por defecto y tú no quieres, aquí es donde se corta.

07
El multiplicador

Composability

Los skills pueden trabajar juntos. Claude puede cargar varios a la vez. Un skill de voz + un skill de formato + un skill de canal = un sistema que trabaja como tú, no como Claude por defecto.
